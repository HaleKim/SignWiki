SignWiki: 청각장애인을 위한 커뮤니티 및 정보 공유 플랫폼 - MVP 제품 요구사항 정의서 (PRD)문서 버전: 1.1작성일: 2025년 10월 28일1. 개요1.1. 제품 비전SignWiki는 한국의 청각장애인과 그 주변인(가족, 친구, 수어 통역사 등)을 위한 통합 커뮤니티 및 정보 공유 플랫폼입니다. 분산된 정책, 복지, 의료, 법률 정보를 한곳에 모으고, 사용자 참여형 수어 데이터 위키를 구축하여 정보 접근성을 혁신적으로 개선하고 활발한 소통의 장을 마련하는 것을 목표로 합니다.1.2. 프로젝트 배경빠르게 변화하는 정책 및 지원 서비스에 대한 정보 비대칭성을 해소하고, 청각장애인 커뮤니티의 실질적인 필요에 부응하는 플랫폼을 신속하게 구축하고자 합니다. 이를 위해 검증된 오픈소스 프로젝트인 mbeps/next_discussion_platform을 기반으로 핵심 기능을 빠르게 개발하고, MVP(Minimum Viable Product)를 통해 핵심 가치를 검증하고자 합니다.1.3. MVP 목표이번 주 발표를 위해, 실제 데이터베이스 연동 없이도 핵심 사용자 경험(UX)을 시연할 수 있는 프론트엔드 중심의 시각적 프로토타입을 구축하는 것을 최우선 목표로 합니다. 사용자는 주요 기능을 눈으로 확인하고 흐름을 이해할 수 있으며, 이를 통해 프로젝트의 비전과 잠재력을 효과적으로 전달합니다.2. 대상 사용자 (User Personas)주요 사용자: 최신 정책, 복지 혜택, 의료 정보 등을 적극적으로 탐색하고자 하는 청각장애인.보조 사용자: 청각장애인 가족, 친구, 수어 통역사, 관련 기관 종사자 등 정보를 공유하고 커뮤니티에 참여하고자 하는 모든 사람.3. MVP 기능 요구사항mbeps/next_discussion_platform의 기본 기능(인증, 게시판 구조 등)을 최대한 활용하며, 아래의 커스텀 기능들을 MVP 범위에 맞춰 구현합니다.3.1. A사 기능 기반 (정보 포털 및 커뮤니티)사용자 기능기능설명MVP 구현 범위1. 홈/정보 열람오늘의 소식, 인기/최신 정보를 확인하고 카테고리별(법률, 의료, 행사 등) 정보를 탐색합니다.**** 홈 화면 및 카테고리별 정보 목록 페이지 UI를 구현합니다. 모든 콘텐츠는 **더미 데이터(Dummy Data)**를 사용하여 표시합니다.2. 지원/복지 서비스정부 지원 사업, 복지 혜택, 신규 정책 목록을 확인하고 간편 신청 및 진행 현황을 조회합니다.**** 서비스 목록 페이지 UI를 구현합니다. '간편 신청' 버튼 클릭 시 나타나는 신청 폼과 '진행 현황' 페이지 레이아웃을 UI만 구현하며, 실제 데이터 제출 및 조회 기능은 포함하지 않습니다.3. 회원 관리SNS(구글, 깃허브 등)를 통한 간편 로그인 및 회원가입을 제공합니다. 접근성(자막, 수어, 글자 크기) 선호도를 설정합니다.**** 기반 프로젝트의 SNS 로그인 기능을 그대로 활용합니다. '접근성 선호 설정'은 마이페이지 내에서 UI 토글(Toggle)만 구현하며, 실제 사이트 전체에 적용되지는 않습니다.4. 정보 등록사용자가 직접 법률, 행사 등 유용한 정보를 등록하고, 자신의 글에 대한 상태(승인 대기 등)를 확인합니다.**** 정보 등록을 위한 글쓰기 에디터 페이지 UI를 구현합니다. '등록 상태 확인' 페이지에서는 정적인 더미 상태 값('승인 대기', '승인' 등)을 표시합니다.5. 마이페이지프로필, 신청 내역, 알림, 포인트 내역 등을 조회하고 환경 설정을 변경합니다.**** 마이페이지의 각 섹션(신청 내역, 알림함 등)에 대한 UI 레이아웃을 구현하고, 모든 데이터는 더미 데이터로 채웁니다.관리자 기능기능설명MVP 구현 범위1~6. 전체 관리자 기능대시보드, 가입자 관리, 콘텐츠 검수, 게시물 관리, 정책 데이터 관리, 로그 감사 등 모든 관리자 기능.**** MVP 범위에서 대시보드, 가입자/콘텐츠 관리 등 주요 관리 페이지의 UI 레이아웃만 구현합니다. 실제 기능 및 데이터 연동은 포함하지 않습니다.3.2. B사 기능 기반 (수어 위키 및 데이터베이스)기능설명MVP 구현 범위1. 수어 자막 위키 & 2. 위키 카테고리'콘텐츠 수어 자막', '종교 경전 수어 자막', '단어 데이터베이스' 세 가지 카테고리의 위키 기능을 제공합니다.**** 세 가지 위키 카테고리 각각에 대한 별도의 페이지 라우팅(/wiki/content, /wiki/religion, /wiki/dictionary)을 부여합니다. 각 페이지는 고유한 레이아웃을 가질 수 있으며, 공통 컴포넌트를 재사용할 수 있습니다. 모든 내용은 더미 데이터로 표시합니다.3. 단어-수형 수어 데이터셋 구축텍스트 자막의 각 단어를 해시태그처럼 분류하고, 사용자가 직접 수형 수어 데이터를 입력/수정할 수 있는 기능을 제공합니다.**** 단어 상세 페이지와 수형 수어 데이터를 입력/수정하는 폼 UI를 구현합니다. 실제 데이터베이스 연동 및 저장 기능은 포함하지 않습니다.4. 웹캠 영상 업로드사용자가 웹캠으로 수어 영상을 촬영하고 업로드하여 데이터베이스에 저장합니다.**** 영상 업로드 및 편집을 위한 UI 레이아웃만 구현합니다. 실제 웹캠 연동 및 영상 처리 기능은 포함하지 않습니다.5. Unity WebGL 연동 기능3D 아바타 컴포넌트 렌더링 및 텍스트-수어 변환 기능 전체.**** 페이지 상단 메뉴바(Header) 영역에 3D 아바타를 켜고 끌 수 있는 On/Off 토글 버튼 UI를 구현합니다. 실제 컴포넌트 연동 및 기능은 포함하지 않습니다.4. MVP 범위 요약 (In-Scope / Out-of-Scope)4.1. In-Scope (포함 범위)A사 요구사항 기반 사용자 기능 및 관리자 기능 전체의 프론트엔드 UI 구현.B사 요구사항 중 WebGL을 제외한 기능들의 프론트엔드 UI 구현.페이지 상단 헤더에 3D 아바타 On/Off 토글 버튼 UI 구현.모든 동적 데이터는 **더미 데이터(Hard-coded JSON 등)**를 사용하여 화면에 표시.mbeps/next_discussion_platform의 사용자 인증(SNS 로그인) 기능 활용.웹캠 업로드 영역 등 기능이 제외된 부분에 대한 자리 표시자(Placeholder) UI 디자인.4.2. Out-of-Scope (제외 범위)Firebase Firestore를 포함한 실제 데이터베이스 연동 및 CRUD 로직 구현.Firebase Cloud Functions 배포 및 관련 서버 자동화 로직 전체.Unity WebGL 컴포넌트의 실제 연동 및 텍스트-수어 변환 기능.웹캠 영상 녹화, 업로드, 데이터화 등 실제 미디어 처리 기능.콘텐츠 승인, 포인트 지급 등 백엔드 비즈니스 로직 전체.5. 기술 스택 및 가정기반 프로젝트: mbeps/next_discussion_platform프론트엔드: Next.js, React, TypeScript, Chakra UI백엔드 (MVP):Firebase Authentication (로그인용): Spark(무료) 요금제에서 사용.Firestore Database (데이터 저장용): Spark(무료) 요금제에서 사용.Firebase Cloud Functions: MVP 범위에서 제외. Blaze(유료) 요금제가 필요하므로 향후 계획으로 이관.가정:mbeps/next_discussion_platform의 기본 인증 및 라우팅 구조는 안정적으로 작동한다.MVP 단계에서는 별도의 디자인 시스템 구축 없이 기반 프로젝트의 Chakra UI를 최대한 활용한다.현재 보유 중인 Unity WebGL 컴포넌트는 MVP 이후 단계에서 React 환경에 성공적으로 통합될 수 있다.6. 성공 지표발표 완료: 이번 주 발표에서 기획된 시나리오에 따라 모든 MVP 기능의 UI 흐름을 성공적으로 시연한다.피드백 수렴: 발표 참여자(이해관계자)로부터 제품 비전에 대한 긍정적인 피드백과 공감대를 형성한다.기술 검증: next_discussion_platform을 기반으로 한 신속한 프로토타이핑 전략이 유효했음을 입증한다.7. 향후 계획 (Post-MVP)Phase 2 (핵심 기능 우선 통합): Unity WebGL 3D 아바타 컴포넌트를 연동하고, 텍스트-수어 변환 기능을 우선 구현합니다. 이는 백엔드 데이터베이스 연동 이전에 프로젝트의 핵심 가치를 조기에 검증하고 시각적으로 보여주기 위함입니다.Phase 3 (백엔드 연동): Firestore 데이터베이스 스키마 설계 및 실제 데이터 연동(CRUD) 기능 구현.Phase 4 (관리자 기능 개발): A사 요구사항 기반의 관리자 대시보드 및 콘텐츠 관리 기능의 실제 로직 개발.Phase 5 (고도화): B사 요구사항 기반의 웹캠 영상 처리 및 집단지성 데이터셋 구축 파이프라인 개발.
